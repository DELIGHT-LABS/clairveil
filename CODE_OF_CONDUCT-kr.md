# 행동 강령

Clairveil contributor는 프로젝트 논의를 집중력 있고, 존중하며, 기술적으로 근거 있는 방식으로 이어가야 합니다.

## 기대하는 행동

- 경험 수준과 전문 영역의 차이를 존중합니다.
- 개인 비판보다 구체적인 기술적 근거를 우선합니다.
- Maintainer가 disclosure를 조율하기 전까지 security-sensitive report는 private하게 유지합니다.
- Review feedback은 실행 가능한 형태로 작성합니다.

## 허용하지 않는 행동

- 괴롭힘, 협박, 차별적 언어
- private key, credential, private vulnerability detail, 개인정보 공개
- 기술 논의를 반복적으로 방해하는 행동

## 집행

Maintainer는 프로젝트와 contributor를 보호하기 위해 issue, pull request, discussion, 기타 project space를 moderate할 수 있습니다. 심각한 위반은 project space 접근 제한으로 이어질 수 있습니다.
